# 操作說明

## 📊 系統概覽

### 進度指示器
頁面上方顯示您的總體進度：
- **圓形進度圖**：綠色進度條顯示完成百分比
- **X/Y Solved**：已完成 / 總題數
- **Score: X/Y**：當前分數 / 總分

### 題目列表
以表格形式顯示所有題目，包含：
- **狀態圖示**：✓ 通過、✗ 失敗、○ 未執行
- **題目名稱**：點擊可開啟詳細頁面
- **得分**：當前得分 / 總分
- **測試通過**：通過的測試案例數量及進度條
- **執行按鈕**：快速執行單一題目的測試

## 🎯 如何使用

### 1. 執行測試
- **執行全部**：點擊「執行全部 (Run All)」按鈕測試所有題目
- **執行單一題目**：
  - 方式一：在列表中點擊題目旁的執行按鈕
  - 方式二：開啟題目詳情頁面後點擊「執行測試 (Run)」

### 2. 編輯程式碼
1. 在題目列表中點擊題目名稱
2. 在右側的編輯器中撰寫程式碼
3. 點擊「儲存 (Save)」保存變更
4. 點擊「執行測試 (Run)」查看結果

### 3. 查看題目說明
- 左側面板顯示題目描述、範例和限制
- **多語言支援**：如果題目有多個語言版本，上方會顯示語言切換按鈕
  - 例如：中文 | English
  - 點擊按鈕即可切換語言

### 4. 查看測試結果
測試執行後，底部會顯示所有測試案例的結果：
- **展開/收合**：點擊案例標題查看詳細資訊
- **INPUT**：測試輸入資料
- **EXPECTED OUTPUT**：預期輸出
- **YOUR OUTPUT**：您的程式輸出（若測試失敗時顯示）

## 📝 狀態說明

- **PASS**：測試通過 ✅
- **FAIL**：答案錯誤 ❌
- **TLE**：執行逾時 ⏱️
- **ERROR**：執行錯誤 ⚠️
- **Compilation Error**：編譯錯誤 🔧

## ⚙️ 評分規則

**重要**：只有當題目的**所有測試案例都通過**時，才會獲得該題目的分數。

## 🚀 提交到 GitHub

完成題目後，點擊「提交 (Push)」按鈕將程式碼推送到 GitHub：
1. 系統會自動 git add、commit 和 push
2. 只會提交 `src/*.cpp` 檔案
3. 確保已設定正確的 Git remote

## 🛠️ 如何新增題目

本系統提供互動式腳本協助您快速新增題目。

### 使用方式
在終端機執行以下指令：
```bash
python3 add_problem.py
```

### 腳本功能
1. **建立檔案結構**：自動建立測資目錄、題目說明檔與程式模板
2. **更新配置**：自動更新系統設定檔
3. **引導式輸入**：只需回答幾個簡單問題（題目名稱、配分等）

### 多語言題目
要建立多語言版本的題目說明：
1. 預設檔案：`problems/pX.md`（中文）
2. 英文版本：`problems/pX.en.md`
3. 其他語言：`problems/pX.[語言代碼].md`

系統會自動偵測並提供語言切換功能。

### 完成後
腳本執行完畢後，您只需要：
1. 編輯 `problems/pX.md` 填寫題目描述
2. 編輯 `src/pX.cpp` 實作解答
3. 在 `tests/pX/inputs/` 和 `tests/pX/outputs/` 目錄下新增測資檔案

## 🌓 深色模式

點擊右上角的主題切換按鈕可在淺色/深色模式間切換。
